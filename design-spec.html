Specification

本頁記錄網站的完整設計規範，包含 Design Tokens、Typography、Spacing、元件庫、動畫與響應式規則。所有修改請以 index.html 的 :root 變數為基準。

01 · Design Tokens

設計代幣

所有顏色皆透過 CSS 變數統一管理。修改主色只需改 --accent 一個值，全站同步更新。

/* index.html <style> 第 60–82 行 */ :root { --accent: #AAE8EC; /* ✏️ 整站主色 */ --accent-dim: rgba(170,232,236,.12); --accent-mid: rgba(170,232,236,.35); --bg: #2b2b2b; /* ✏️ 頁面背景 */ --bg2: #232323; /* 次要背景 */ --text: #ffffff; --text-dim: rgba(255,255,255,.55); --text-muted: rgba(255,255,255,.28); --border: rgba(170,232,236,.18); --border-subtle: rgba(255,255,255,.06); --card: rgba(255,255,255,.04); }

05 · Animation

動畫規則

元件進場以 fadeUp 為主，搭配 stagger delay 製造層次感。互動 hover 使用短時 transition。

Keyframes

/* fadeUp · 大部分元件的進場動畫 */ @keyframes fadeUp { from { opacity: 0; transform: translateY(20px); } to { opacity: 1; transform: translateY(0); } } /* growBar · 問卷長條圖動畫 */ @keyframes growBar { to { width: var(--w); } }

Stagger Pattern（進場延遲）

Element 1 · delay 0s
Element 2 · delay .15s
Element 3 · delay .30s

06 · Responsive

響應式規則

三個斷點：Desktop（>1024px）· Tablet（≤1024px）· Mobile（≤768px）· Mobile-SM（≤480px）。

Breakpoint 主要變化
>1024px
Desktop
  • Section padding LR: 80px
  • UJM: 桌面五欄表格模式
  • Persona / Flow: 兩欄並排
≤1024px
Tablet
  • Section padding LR: 40px
  • Footer padding: 32px 40px
≤768px
Mobile
  • Nav padding: 0 12px
  • Cards grid: repeat(2, 1fr)
  • HMW、Pain grid: 1fr
  • Footer: 垂直排列，置中
  • UJM: 切換為 slider 模式
  • Direction grid: 1fr 1fr
≤480px
Mobile SM
  • Section padding LR: 10%
  • Cards grid: 1fr（單欄）
  • Direction grid: 1fr
  • Persona header: 垂直排列
